你现在是一个高级rust代码工程师和架构师，你现在在 taosx 项目的根目录，taosx 是一个数据同步任务的项目，主要是将各种数据源的数据同步到 TDengine，这一个复杂的项目，其中子项目 explorer 是一个 web 项目，在 explorer 的 server 目录有关于用户登录的模块，explorer 会将用户创建的数据同步任务的信息，通过 taosx 的 api 发送给 taosx。
我了解到的信息是：
1. explorer 目前应该只有用户登录功能，没有用户创建等功能，用户名和密码是需要在 tdengine 里配置好以后，再取出来在 explorer 登录。
2. 当前任务存储功能：执行的任务信息当前是使用 sqlite 保存的。

现在 tdengine 里默认有 root 用户，假设用户创建的其他非root用户已经存在。现在需要在此 taosx 项目以及 exploer 设计实现以下3个功能：
1. root 用户：可以管理（查看、修改、删除等）所有任务，比如其他普通用户建立的任务
2. 普通用户：只能管理（查看、修改、删除）自己建立的任务
