

rust 4 种宏, 表达能力都不一样：

1. macro_rules: 声明宏，简单的模块内使用的宏
2. `#proc_macro` : 函数宏，类似 vec! 的宏，直接产生过程的执行代码，扩展性最强，形式上像个函数，用在代码模块里
3. 属性宏 `#proc_macro_attribute`：可以带属性的，第一个参数是属性，主要用在函数上，类似装饰器，可以带参数，实现类似 AOP 的概念
4. 派生宏``#proc_macro_derive``：用于继承 trait 之类的，主要用在 struct 上，产生新的方法，必须使用 #derive() 宏, 且在结构体上

其实 属性宏 和 派生宏 最后用 parse_macro_input! 都转为内置的类型。

函数宏用起来像函数，声明宏 反而和 函数宏 长的更像，但是本质是不一样的！



| 宏类型                    | 适用场景                              | 复杂度 |
| ------------------------- | ------------------------------------- | ------ |
| `macro_rules!`            | 简单代码生成、DSL                     | 低     |
| `#[proc_macro]`           | 复杂 DSL、自定义语法                  | 中     |
| `#[proc_macro_attribute]` | AOP（日志、路由、权限）、包装函数     | 中     |
| `#[proc_macro_derive]`    | 代码生成（Builder、Serialize、Clone） | 高     |



**关键依赖**：

- `syn`：解析 Rust 语法树
- `quote`：生成 Rust 代码模板
- `proc-macro2`：处理 `TokenStream` 的工具库
