部分AI 输出的内容：

```
  根本原因：

  // 错误代码
  if bootstrap_peer_id.map(|id| id != peer_id).unwrap_or(true) && !connected_to_b {
      // 此时 bootstrap_peer_id 还是 None，unwrap_or(true) 返回 true
      // 误判 Bootstrap 为 Node B！
  }

  修复后的正确逻辑

  ┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
  │  Connection     │────▶│  Identify       │────▶│  Send Request   │
  │  Established    │     │  Received       │     │  (only to NodeB)│
  └─────────────────┘     └─────────────────┘     └─────────────────┘
          │                        │                       │
          │                        ▼                       │
          │            ┌─────────────────┐                │
          │            │  Is Bootstrap?  │─Yes────────────┘ (skip)
          │            └─────────────────┘                │
          │                   │ No                        │
          │                   ▼                           │
          │            ┌─────────────────┐                │
          │            │  Is Node B?     │─Yes────────────▶ (send)
          │            └─────────────────┘                │
          │                       │                       │
          └───────────────────────┘ (wait for identify)

  1. ConnectionEstablished - 只记录连接，不发送请求
  2. Identify::Received - 识别对方身份：
    • 如果是 Bootstrap → 记录 bootstrap_peer_id
    • 如果是 Node B → 记录 identified_node_b，检查是否已连接，是则发送请求，否则拨号
  3. Kademlia 发现 - 自动拨号新发现的 peer

  使用步骤

  确保启动顺序：

  # 机器 1：启动 Bootstrap
  ./target/debug/bootstrap --port 9090 --secret-key-seed 0

  # 机器 2：启动 Node B（连接到 Bootstrap）
  ./target/debug/node_b 10001 8080 "/ip4/<bootstrap_ip>/tcp/9090/p2p/<bootstrap_peer_id>"

  # 机器 1：启动 Node A（连接到 Bootstrap，自动发现 Node B）
  ./target/debug/node_a 10002 "/ip4/192.168.126.85/tcp/9090/p2p/<bootstrap_peer_id>"

  现在 Node A 会：

  1. 连接 Bootstrap
  2. 通过 Kademlia 发现 Node B
  3. 自动拨号 Node B
  4. 识别 Node B 后发送请求
```
