

pnpm 安装：通过 nvm 安装

```
curl -fsSL https://fnm.vercel.app/install | bash         # 地址不可用

echo 'eval "$(fnm env --use-on-cd)"' >> ~/.bashrc
source ~/.bashrc

fnm install 24
fnm default 24
fnm use 24

```

通过 cargo 安装 fnm :

```
# 安装 fnm
cargo install fnm

# 添加到 PATH（cargo 默认安装到 ~/.cargo/bin）
echo 'export PATH="$HOME/.cargo/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(fnm env)"' >> ~/.bashrc
source ~/.bashrc

fnm install 24
fnm default 24
fnm use 24

npm install -g pnpm
```



```
# 运行全部用例（推荐入口）
pnpm -C explorer exec playwright test

# 指定 baseURL（CI/远端/非默认端口）
PLAYWRIGHT_BASE_URL=http://127.0.0.1:6060 pnpm -C explorer exec playwright test

# 本地调试：复用已有登录态，不重复 globalSetup
PLAYWRIGHT_SKIP_GLOBAL_SETUP=true pnpm -C explorer exec playwright test

# 仅跑某个 spec
pnpm -C explorer exec playwright test tests/login.spec.ts

# UI 模式 / Debug
pnpm -C explorer exec playwright test --ui
pnpm -C explorer exec playwright test --debug
```

命令不生效原因:

最可能的原因是：**文件名不符合 Playwright 的默认测试匹配模式**。

Playwright 默认只识别 `*.spec.ts` 或 `*.test.ts` 结尾的文件，而你的文件是 `zgc.login.ts`，所以被忽略了。



使用:

```
PLAYWRIGHT_BASE_URL=http://192.168.2.158:6060 pnpm exec playwright test tests/zgc.login.spec.ts --ui


PLAYWRIGHT_BASE_URL=http://192.168.2.158:6060 pnpm exec playwright test tests/zgc.login.spec.ts --debug

PLAYWRIGHT_BASE_URL=http://192.168.126.85 pnpm exec playwright test tests/tmq-task.spec.ts --ui
```



跳过登录态:

```
// 第一次：完整运行（执行 globalSetup 登录）
pnpm -C explorer exec playwright test tests/zgc.login.spec.ts

// 修改代码后再次调试：跳过登录，直接测试（快！）
PLAYWRIGHT_SKIP_GLOBAL_SETUP=true pnpm -C explorer exec playwright test tests/zgc.login.spec.ts

// 或者带 UI 调试（更常用）
PLAYWRIGHT_SKIP_GLOBAL_SETUP=true pnpm -C explorer exec playwright test --ui
```
